# 문서 아카이브 (Document Archive)

> **과거 버전 및 참고용 문서 보관소**

**목적**: 프로젝트 진행 중 변경되거나 대체된 문서를 보관하여 이력 추적 및 참고용으로 사용

---

## 📋 아카이브 정책

### 보관 대상
1. **대체된 설계 문서**: 새로운 설계로 교체된 이전 버전
2. **폐기된 아이디어**: 채택되지 않은 설계안
3. **과거 버전**: 주요 개선 전의 문서 스냅샷

### 보관하지 않는 것
- 현재 유효한 문서 (이전 버전은 Git history로 추적)
- 일시적인 메모나 초안

---

## 📁 현재 아카이브 상태

### 보관된 문서 없음
현재 모든 문서가 최신 상태입니다.

**최초 헌법 제정**: 2026-01-07

## 🔄 아카이브 프로세스

### 문서를 아카이브할 때
1. 파일을 `docs/history/`로 이동
2. 파일명에 날짜 추가: `{원본파일명}_archived_{YYYY-MM-DD}.md`
3. 이 README.md에 아카이브 기록 추가
4. 원본 위치에 아카이브 안내 주석 추가 (필요 시)

### 예시
```bash
# 기존 파일
docs/architecture/old-design.md

# 아카이브
docs/history/old-design_archived_2026-01-07.md
```

## 🔍 아카이브 문서 찾기

### Git History로 과거 버전 확인
```bash
# 특정 파일의 모든 변경 이력
git log --follow docs/architecture/blog-architecture-context.md

# 특정 시점의 파일 내용 보기
git show {commit-hash}:docs/architecture/blog-architecture-context.md
```
